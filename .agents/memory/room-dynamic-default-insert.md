---
name: Room dynamic-default INSERT trap
description: Room cannot express System.currentTimeMillis() as a SQLite DEFAULT; raw execSQL must supply these columns explicitly or INSERT OR IGNORE silently drops every row.
---

## The rule
Any `WordEntity` column whose Kotlin default is `System.currentTimeMillis()` (`created_at`, `last_accessed`) gets generated by Room as `INTEGER NOT NULL` with **no SQLite DEFAULT clause**. A raw `execSQL` INSERT that omits those columns triggers a NOT NULL constraint violation, which `INSERT OR IGNORE` swallows silently — no exception is thrown, nothing is inserted, but any `inserted++` counter that fires after `execSQL` still increments.

**Why:** Room translates static Kotlin defaults (0, "", false) to SQLite DEFAULT literals. Dynamic expressions like `System.currentTimeMillis()` cannot be expressed as a constant SQL literal, so Room emits no DEFAULT at all.

**How to apply:**
- Always include `created_at` and `last_accessed` in `DatabaseSeeder.INSERT_SQL` and pass `System.currentTimeMillis()` (captured once before the loop) as bind parameters.
- To get an accurate "actually inserted" count from INSERT OR IGNORE, query `SELECT changes()` immediately after each `execSQL` call (before the next statement).
- If more NOT NULL columns without static defaults are added to `WordEntity`, add them to the INSERT too.
